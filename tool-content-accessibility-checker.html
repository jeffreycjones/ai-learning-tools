Accessibility Best Practices

H

Use Proper Heading Hierarchy

Structure content with H1 for main title, H2 for major sections, and H3+ for subsections. Never skip heading levels, as this breaks the logical structure for screen readers.

Link

Write Descriptive Link Text

Avoid generic phrases like "click here" or "read more." Use clear, descriptive text that explains where the link goes, such as "View assignment rubric" or "Download syllabus."

ABC

Keep Reading Level Accessible

Aim for a grade 8-10 reading level. Use shorter sentences, simpler words, and break up long paragraphs. Tools like Hemingway Editor can help evaluate readability.

Clear

Use Plain Language

Replace jargon with everyday terms. Define necessary technical terms. Use active voice. Be direct and concise. This benefits all learners, especially those with cognitive disabilities.

Table

Structure Tables with Headers

Always include header rows and columns. Mark headers as such in the code. Keep tables simple with clear relationships. Avoid using tables for layout purposes.

Color

Avoid Color-Only Information

Never convey information through color alone. Use color plus text, icons, or patterns. Ensure 4.5:1 contrast ratio for normal text and 3:1 for large text (WCAG AA).
